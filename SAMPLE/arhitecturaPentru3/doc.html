Introduction

AttendanceWebTracker is an application that allows online presence on activities associated with an academic program - for example, course / laboratory / seminar classes for each discipline or participation in a contest. The tool will support both participants (ie students) and people who want the presence (eg, lab, event organizer). Additionally, people present can be awarded scores and / or associate content terms (tags) or notes. To appeal to an effective method of preventing false presence. Generic, through data viewing techniques, intermediate or final reports on the presence and scores accumulated by an individual / group, including statistics, comparisons, multi-criteria filtering, etc. will be generated. The data will be imported from an external web storage source and exported as JSON and HTML5 + documents and related microdata schemes.

Create a new session

In order to create a new event the organizer can click on "Create a new session"

After the user clicks the button, ann popUp will appear. In this popUp are three input fields and two buttons:

  • Enter title - here must be filled with the title of the event that will be created
  • Enter max students - here will be filled with the maximum number of participants that can be present at the event
  • Session code - a code that will shared to the participants in order to let them mark themselfs present at the event
  • create code button - will create a random session code
  • Create button - will create the event

Create Chart

In order to create a chart view of some data, the organizer can click on "Create Chart"

After the user clicks the button, ann popUp will appear. In this popUp are some input fields used by the organizer to filter the data used to create a chart view:

  • First field is a select field. In this field, the organizer can select between some predefined charts in order to see them
  • The next two fields are used to select grades that are between the input from "min grade" field and "max grade" field
  • The fields "start time" and "end time" are used to filter the data based on the time when the students marked themselfs as present at one event
  • The "Select just passed students" checkbox is used by the organizer to see data just from the passed students
  • All the following checkboxes are used by the organizer to see just data from the selected sessions(the sessions that he can check are represented by events organised by him )
  • Show filtered chart button - will create and display a chart view based on the filters selected by the organizer

Import

In order to import some data and to store the into the database, the organizer can click on the "import" button. After he clicks the button the application will search for csv files from the google drive account of the logged in user. Then a popUp appears with some fields that the organizer needs to fill in order to import data.

The fields are:

  • Select the file - is a select field where are displayed all the csv files from the google drive account of the logged in user
  • Select for which session you upload the file - is a select field in which the organizer needs to select for which event he wants to import data
  • Import button - will perform the import action

Export

In order to export some data the organizer can click on the "export" button. After he clicks the button a popUp appears with two checkboxes that the organizer needs to fill in order to export data.

The checkboxes are:

  • JSON - if checked, the export button will download a json file
  • Html - if checked, the export button will download a html file
  • If both are checked, the export button will download a html file and a json file
  • Export button - will perform the import action

Motivation(how our application helps in real world)

Teachers need a smart way to keep track of courses, grades and attendance. It is a lot easier to just create a class on our app and ask the students to enter there and mark themselfs as present. In this way, the teacher can give grades and feedback to every student. Beside that, this Attendance Web Tracker is safe because a student can mark himself present based on his geolocation ,unlike the old fashion way where students have to write their name on a piece of paper because they canr write as many names as they like.
